Этап 1: Инициализация и Базовые Стили (Mobile First)
 * [ ] 1. Установить viewport meta tag.
 * [ ] 2. Подключить Normalize.css или написать свой CSS Reset.
 * [ ] 3. Установить box-sizing: border-box; глобально.
 * [ ] 4. Определить базовый font-size для html (например, 100% = 16px).
 * [ ] 5. Задать основные стили для body (шрифт, line-height, цвет текста) для мобильного вида.
 * [ ] 6. Разработать типографическую шкалу (h1-h6, p, small) в rem для мобильного вида.
 * [ ] 7. Сверстать и стилизовать основные компоненты (кнопки, поля ввода, карточки) для мобильного вида, используя относительные единицы.
 * [ ] 8. Реализовать мобильную версию навигации (например, "бургер").
Этап 2: Основной макет и Контент (Mobile First)
 * [ ] 9. Сверстать структуру страницы (шапка, основной контент, подвал) для мобильного вида.
 * [ ] 10. Создать основной .container с боковыми padding для мобильных.
 * [ ] 11. Расположить все блоки контента в одну колонку для мобильных.
 * [ ] 12. Убедиться, что изображения и медиа гибкие (max-width: 100%; height: auto;).
Этап 3: Добавление Брейкпоинтов и Адаптация
 * [ ] 13. Первый брейкпоинт (например, min-width: 768px - планшеты):
   * [ ] При необходимости, немного увеличить базовый font-size на html или размеры ключевых текстовых элементов.
   * [ ] Адаптировать .container (увеличить padding, если нужно; max-width еще может не применяться).
   * [ ] Перестроить сетки (например, с 1 колонки на 2-3 колонки с помощью Flexbox/Grid).
   * [ ] Адаптировать навигацию (может оставаться "бургером" или трансформироваться).
   * [ ] Проверить отступы между элементами и секциями.
 * [ ] 14. Второй брейкпоинт (например, min-width: 1024px - ноутбуки/малые десктопы):
   * [ ] Возможно, еще немного увеличить шрифты.
   * [ ] Задать max-width для .container и центрировать его.
   * [ ] Перестроить сетки для большего количества колонок, если необходимо.
   * [ ] Реализовать десктопную навигацию (горизонтальное меню).
   * [ ] Проверить длину строк текста (не более 75 символов).
 * [ ] 15. Третий брейкпоинт (например, min-width: 1366px или 1440px - большие десктопы), если требуется:
   * [ ] Проверить, как смотрится макет с max-width контейнера. Возможно, потребуется увеличить max-width или адаптировать фоны/широкие секции.
   * [ ] Убедиться, что все элементы хорошо масштабируются и нет пустующих областей или перегруженности.
Этап 4: Тестирование и Отладка
 * [ ] 16. Инструменты разработчика в браузере:
   * [ ] Постоянно использовать режим адаптивного дизайна (Responsive Design Mode) для проверки на разных разрешениях.
   * [ ] Проверять перестроение макета при плавном изменении ширины окна.
 * [ ] 17. Кроссбраузерность: Тестировать в последних версиях Chrome, Firefox, Safari, Edge.
 * [ ] 18.Реальные устройства: Обязательно тестировать на реальных смартфонах (iOS, Android) и планшетах, если есть доступ. Обращать внимание на удобство касания (tap targets мин. 44-48px).
 * [ ] 19. Ориентация экрана: Проверить отображение в портретной и альбомной ориентации.
 * [ ] 20. Валидация: Проверить HTML и CSS на валидность.
 * [ ] 21. Доступность (Accessibility): Проверить навигацию с клавиатуры, контрастность.
 * [ ] 22. Производительность: Использовать Lighthouse для анализа скорости загрузки и других метрик. Оптимизировать изображения, минифицировать CSS/JS.
